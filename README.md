# Cplusplus-Program
# CS 210 Portfolio Reflection – Project Three: Corner Grocer Tracking Program

# Project Summary
For this project, I developed a C++ application for the Corner Grocer that tracks the frequency of grocery items purchased throughout the day. The program reads data from an input file, counts how many times each item appears, and allows the user to search for a specific item, display all item frequencies, view a histogram, and automatically create a backup file containing the frequency data. The goal of the project was to help the store analyze purchasing trends and organize its produce section more effectively.

# What I Did Particularly Well
One aspect I believe I did well was organizing the program using object-oriented programming principles. I separated the code into multiple files and created an 'ItemTracker' class to manage the program's functionality. I also used a C++ map to efficiently store and retrieve item frequencies. This made the program easier to understand, maintain, and expand.

# Areas for Improvement
If I were to continue improving this project, I would add more input validation and error handling, such as checking whether the input file exists before attempting to read it and allowing users to search for items regardless of uppercase or lowercase letters. I would also improve the user interface by creating a cleaner menu layout and providing more informative error messages. These changes would make the program more user-friendly, secure, and reliable.

# Challenges and Resources
The most challenging part of this project was working with file input and output and using maps to store frequency data. I also had to troubleshoot issues related to reading the input file correctly in Visual Studio. I overcame these challenges by reviewing the course materials, using the zyBooks textbook, referring to my instructor's resources, and practicing debugging techniques. These resources will continue to be valuable as I complete future programming projects.

# Transferable Skills
This project strengthened several skills that will be useful in future courses and software development projects. These include object-oriented programming, creating and using classes, working with maps, reading and writing files, implementing menus, using functions to organize code, and debugging programs. These are all fundamental programming skills that apply to many different types of software applications.

# Maintainability, Readability, and Adaptability
I designed the program to be maintainable by separating responsibilities into different source and header files and using descriptive function and variable names. I included inline comments to explain important sections of the code and kept the program modular so that additional features could be added without significantly changing the existing code. This approach improves readability, simplifies debugging, and makes the program easier to maintain as requirements change.
